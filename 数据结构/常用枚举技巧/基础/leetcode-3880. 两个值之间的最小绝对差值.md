# [3880. 两个值之间的最小绝对差值](https://leetcode.cn/problems/minimum-absolute-difference-between-two-values/)

[简单]

给你一个只包含 0、1 和 2 的整数数组 `nums`。

如果 `nums[i] == 1` 且 `nums[j] == 2`，则称下标对 `(i, j)` 为 **有效** 的。

请返回所有有效下标对中 `i` 和 `j` 之间的 **最小** 绝对差。如果不存在有效下标对，则返回 -1。

下标 `i` 和 `j` 之间的绝对差定义为 `abs(i - j)`。

**示例 1：**

**输入：** nums = [1,0,0,2,0,1]

**输出：** 2

**解释：**

有效下标对有：

- (0, 3)，其绝对差为 `abs(0 - 3) = 3`。
- (5, 3)，其绝对差为 `abs(5 - 3) = 2`。

因此，结果是 2。

**示例 2：**

**输入：** nums = [1,0,1,0]

**输出：** -1

**解释：**

数组中不存在有效下标对，因此结果是 -1。

**提示：**

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 2`

```python
class Solution:
    def minAbsoluteDifference(self, nums: list[int]) -> int:
        res = 101
        n = len(nums)
        one =inf
        two =inf
        for i in range(n):
            if nums[i]==1:
                one = i
            if nums[i]==2:
                two = i
            res =min(res,abs(one-two))
        return res if res <101 else -1
```

## 算法思想：在线维护最近出现位置

### 1. 核心思路

- 使用两个变量 `one` 和 `two` 分别记录**最近一次遇到的值为 1 的索引**和**最近一次遇到的值为 2 的索引**。
- 遍历数组，每当遇到 `1` 或 `2` 时，立即更新对应的索引，然后计算当前 `abs(one - two)`，并尝试更新全局最小值 `res`。
- 由于索引是单调递增的，**最小距离一定出现在某个 1 和某个 2 “相邻出现”的时刻**（这里的相邻是指：在遍历过程中，一个刚出现，另一个是之前最近出现的那一个）。因此，维护最近位置就足够了，无需记录所有位置。

### 2. 为什么正确？

假设最优的一对 `1` 和 `2` 分别位于索引 `i`（`1`）和索引 `j`（`2`），且 `i < j`。
在遍历到 `j` 时，`one` 可能等于 `i`，也可能大于 `i`（如果 `i` 之后还有其他的 `1`）。但无论哪种情况，`one` 都至少是最近的一个 `1`，其索引 ≥ `i`，因此 `abs(one - two)` = `j - one` ≤ `j - i`。同理，如果 `i > j` 也类似。所以每次用最近的一对计算，必然能捕捉到最小差值。

### 3. 复杂度

- **时间复杂度**：O(n)，一次遍历。
- **空间复杂度**：O(1)，只用了常数个变量。

### 4. 初始化哨兵值的解读

- `one = inf` 和 `two = inf`：在未遇到 `1` 或 `2` 时，保证 `abs(one-two)` 的结果是无穷大，不会干扰 `min` 的计算。
- `res = 101`：充当一个**大于任何可能答案的初始值**。该写法隐含假设数组长度 ≤ 100，所以最大索引差不超过 99，`101` 作为“不可能的大值”是安全的。但更通用的写法应使用 `res = float('inf')`，以避免数组长度可能超过 101 时的逻辑错误。

### 5. 局限性 / 潜在问题

- 若数组长度 ≥ 102，且最小差值为 100，那么 `res` 的初始值 101 可能无法被任何合法差值（≤100）打败，导致 `res` 依然为 101，最后会错误返回 `-1`。
  **改进**：直接用 `res = float('inf')`，返回时判断 `return res if res != float('inf') else -1`。

**总结**：这是一个**双哨兵在线更新**的贪心算法，利用“最小距离必然发生在最近一对出现时”的性质，以 O(n) 时间高效解决问题。唯一需要注意的是初始最大值应足够大。
