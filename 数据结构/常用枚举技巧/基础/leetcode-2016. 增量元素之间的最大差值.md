# [2016. 增量元素之间的最大差值](https://leetcode.cn/problems/maximum-difference-between-increasing-elements/)

给你一个下标从 **0** 开始的整数数组 `nums` ，该数组的大小为 `n` ，请你计算 `nums[j] - nums[i]` 能求得的 **最大差值** ，其中 `0 <= i < j < n` 且 `nums[i] < nums[j]` 。

返回 **最大差值** 。如果不存在满足要求的 `i` 和 `j` ，返回 `-1` 。

 

**示例 1：**

```
输入：nums = [7,1,5,4]
输出：4
解释：
最大差值出现在 i = 1 且 j = 2 时，nums[j] - nums[i] = 5 - 1 = 4 。
注意，尽管 i = 1 且 j = 0 时 ，nums[j] - nums[i] = 7 - 1 = 6 > 4 ，但 i > j 不满足题面要求，所以 6 不是有效的答案。
```

**示例 2：**

```
输入：nums = [9,4,3,2]
输出：-1
解释：
不存在同时满足 i < j 和 nums[i] < nums[j] 这两个条件的 i, j 组合。
```

**示例 3：**

```
输入：nums = [1,5,2,10]
输出：9
解释：
最大差值出现在 i = 0 且 j = 3 时，nums[j] - nums[i] = 10 - 1 = 9 。
```

**提示：**

- `n == nums.length`
- `2 <= n <= 1000`
- `1 <= nums[i] <= 109`

```python
class Solution:
    def maximumDifference(self, nums: List[int]) -> int:
        res = 0
        min_nums = nums[0]
        for j in nums:
            res = max(j-min_nums,res)
            min_nums = min(min_nums,j)
    
        return res or -1
```

### 1. 核心思路

- 维护一个变量 `min_nums`，表示**当前已遍历过的元素中的最小值**（相当于最佳买入点）。
- 遍历数组时，对于当前元素 `j`（相当于卖出价格），计算它和 `min_nums` 的差值，并尝试更新最大差值 `res`。
- 每步都用 `min()` 更新 `min_nums`，保证它总是历史最低点。

由于遍历顺序是从左到右，**天然保证了买入索引小于卖出索引**（因为最小值来自之前的元素）。

### 2. 为什么能正确处理无解情况？

- 若数组**不存在严格递增的二元组**（即整个数组是非递增的，如 `[3,2,1]` 或 `[2,2,2]`），则在任何位置 `j` 都不会大于它前面的历史最小值，所以 `j - min_nums ≤ 0`。
  `res` 初始为 `0`，每次 `max` 后仍保持 `0`。
- 最后返回 `res or -1`：因为 `0` 是假值，逻辑或会返回 `-1`，恰好满足题目要求。

### 3. 正确性证明

设 `min_idx` 为已遍历部分的最小值所在索引，`j` 为当前索引。显然 `min_idx ≤ j`（因为 `min_nums` 在遇到当前元素之前就已更新或等于当前元素）。

- 若 `nums[j] > min_nums`，则差值正，是一个可行解，`res` 会更新为历史最大差值。
- 若 `nums[j] ≤ min_nums`，则以当前元素作为卖出点无利润，但我们会更新 `min_nums`（若它是新的最小值），为后面的元素提供更低的买入点。

这种“最低点买入，之后最高点卖出”的贪心策略能覆盖所有可能的最优交易对，因为最优解中的买入点一定是某个前缀的最小值。

### 4. 复杂度

- 时间复杂度：**O(n)**，只需一次遍历。
- 空间复杂度：**O(1)**，仅使用常数变量。
