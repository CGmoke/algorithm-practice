# [2657. 找到两个数组的前缀公共数组](https://leetcode.cn/problems/find-the-prefix-common-array-of-two-arrays/)

难度[中等] 	相关标签：高级工程师	位运算 	数组	哈希表

给你两个下标从 **0** 开始长度为 `n` 的整数排列 `A` 和 `B` 。

`A` 和 `B` 的 **前缀公共数组** 定义为数组 `C` ，其中 `C[i]` 是数组 `A` 和 `B` 到下标为 `i` 之前公共元素的数目。

请你返回 `A` 和 `B` 的 **前缀公共数组** 。

如果一个长度为 `n` 的数组包含 `1` 到 `n` 的元素恰好一次，我们称这个数组是一个长度为 `n` 的 **排列** 。

**示例 1：**

```
输入：A = [1,3,2,4], B = [3,1,2,4]
输出：[0,2,3,4]
解释：i = 0：没有公共元素，所以 C[0] = 0 。
i = 1：1 和 3 是两个数组的前缀公共元素，所以 C[1] = 2 。
i = 2：1，2 和 3 是两个数组的前缀公共元素，所以 C[2] = 3 。
i = 3：1，2，3 和 4 是两个数组的前缀公共元素，所以 C[3] = 4 。
```

**示例 2：**

```
输入：A = [2,3,1], B = [3,1,2]
输出：[0,1,3]
解释：i = 0：没有公共元素，所以 C[0] = 0 。
i = 1：只有 3 是公共元素，所以 C[1] = 1 。
i = 2：1，2 和 3 是两个数组的前缀公共元素，所以 C[2] = 3 。
```

**提示：**

- `1 <= A.length == B.length == n <= 50`
- `1 <= A[i], B[i] <= n`
- 题目保证 `A` 和 `B` 两个数组都是 `n` 个元素的排列。

```python
class Solution:
    def findThePrefixCommonArray(self, A: List[int], B: List[int]) -> List[int]:
        res = []
        n =len(A)
        setA=set()
        setB=set()
        for  i in range(n):
            setA.add(A[i])
            setB.add(B[i])
            temp =len(setA & setB)
            res.append(temp)
        return res
```

### 核心思路

题目要求：
对于每个前缀长度 `i+1`，统计 `A[0..i]` 和 `B[0..i]` 中**共有**的元素个数，结果存入数组返回。

一种直接想法是每次截取前缀，然后求交集大小。但这会重复扫描，时间复杂度 O(n²)。
本解法通过**动态维护两个集合**，每次只需 O(1) 时间即可获得交集大小，总体降到 O(n)。

### 算法步骤

1. 初始化两个空集合 `setA` 和 `setB`，以及结果列表 `res`。
2. 从 `i = 0` 到 `n-1` 遍历：
   - 将 `A[i]` 加入 `setA`，将 `B[i]` 加入 `setB`。
   - 计算当前两个集合的**交集大小** `len(setA & setB)`，这就是 `A[0..i]` 与 `B[0..i]` 的公共元素个数。
   - 将该值追加到 `res`。
3. 遍历结束后返回 `res`。

### 正确性证明

- 第 `i` 步时，`setA` 包含 `A` 前 `i+1` 个元素，`setB` 包含 `B` 前 `i+1` 个元素。
- 两个集合的交集恰好包含了同时在 `A[0..i]` 和 `B[0..i]` 中出现过的所有元素，因此交集的大小就是公共元素的数量。
- 由于集合的添加和交集计算都是对无重复元素的，所以即使数组中有重复（虽然题目说明是排列，无重复），算法依然正确。

### 为什么高效？

- 集合的 `add` 和取交集 `&` 都是平均 O(1) 时间复杂度（实际与元素数量相关，但这里每个元素只加入一次，所以整体是 O(n)）。
- 如果每次都用切片重建集合再求交集，总复杂度会达到 O(n²)，在大数据量下会超时。本解法则通过增量更新的方式，完美避免了重复计算。

**时间复杂度**：O(n)。

##### **空间复杂度**：O(n)，用于存储两个集合。
