# 灯塔

##### （难度: 中LV.8  标签: 国赛, 动态规划, 2025）

### 问题描述

在一片海岸线上，有 n*n* 个灯塔排成一排，从左到右编号为 11 到 n*n*。你需要点亮其中的一些灯塔以指引船只航行。给定一个长度为 m*m* 的点亮序列 a1,a2,…,am*a*1,*a*2,…,*a**m*，其中第 i*i* 个操作表示尝试点亮编号为 ai*a**i* 的灯塔。然而为了节约能源，如果灯塔 ai−1*a**i*−1 或灯塔 ai+1*a**i*+1 已经被点亮，则灯塔 ai*a**i* 无法被点亮，该操作将被跳过。当然，同一个灯塔只会被点亮一次。

你可以从给定的点亮序列中挑选一个子序列（保持操作的相对顺序），按照子序列的顺序依次执行点亮操作。请问，最多能成功点亮多少个灯塔？

### 输入格式

输入的第一行包含两个正整数 n,m*n*,*m* ，用一个空格分隔。

第二行包含 m*m* 个正整数 a1,a2,…,am，相邻整数之间使用一个空格分隔。

### 输出格式

输出一行包含一个整数表示答案。

### 样例输入

```text
10 9
1 3 2 5 2 10 9 6 5
```

### 样例输出

```text
4
```

### 样例说明

其中一种方案：保留子序列 1,3,5,91,3,5,9，可以点亮四个灯塔。

```python
n, m = map(int, input().split())          # 读取 n 和 m，但 n 实际上未使用
numbers = list(map(int, input().split())) # 读取长度为 m 的操作序列
numbers = sorted(set(numbers))            # 去重并排序，得到所有出现过的灯塔编号（升序）
dp = [0] * len(numbers)                   # dp[i] 表示前 i+1 个编号中最多能选多少个
for i in range(len(numbers)):
    if i == 0:
        dp[i] = 1                         # 只有一个灯塔时，肯定可以点亮
    else:
        # 判断当前编号与前一个编号是否相邻（数值差 1）
        if numbers[i] == numbers[i-1] + 1:
            # 相邻：不能同时选当前和上一个，只能选其中一个
            if i >= 2:
                # 经典“打家劫舍”转移：要么不选当前（dp[i-1]），要么选当前并加上 i-2 的结果
                dp[i] = max(dp[i-1], dp[i-2] + 1)
            else:
                # i == 1 时，只有两个相邻灯塔，最多只能选 1 个
                dp[i] = dp[i-1]           # 即 dp[0] = 1
        else:
            # 不相邻：可以同时选当前灯塔和前一个状态的所有灯塔
            dp[i] = dp[i-1] + 1
print(dp[-1])                             # 输出最后一个 dp 值，即最多能点亮的灯塔数
```

解决一个简化后的问题：在给定的点亮序列中，每个灯塔只需要考虑一次（去重），并且可以任意安排点亮顺序（忽略原序列的顺序），目标是选出尽可能多的灯塔，使得它们两两不相邻（即编号差的绝对值不等于1）。代码使用了动态规划（DP）来求解这个“最大独立集”问题。

### 动态规划的逻辑

- 经过排序后，编号序列中相邻的项如果数值差 1，表示它们在海岸线上是相邻的灯塔，不能同时被点亮。
- 当遇到两个编号不连续时，可以安全地将当前灯塔加入集合（总数 +1）。
- 当遇到连续编号时，类似“打家劫舍”问题：不能同时抢相邻两家，只能取前一个状态（不选当前）或者取前两个状态加上当前（选当前）。
