回溯算法，本质上是一种**在解空间树中深度优先搜索**的策略，目的是找出所有满足约束条件的解。可以把它理解为一种**带撤销功能的、聪明的穷举**。

------

### 一、核心思想：逐步构建与及时撤销

回溯的核心是试探性地向前走一步，如果发现此路不通或已经得到一个完整解，就**撤销（回溯）**这一步，再尝试其他可能。

这对应到代码模板中非常固定的三段式结构：

1. **做出选择**：将当前节点加入路径。
2. **递归探索**：基于该选择，进入下一层决策。
3. **撤销选择**：从路径中移除该节点，回到之前的状态，以便尝试下一个选择。

------

### 二、三个核心要素

任何回溯问题，都可以通过回答下面三个问题来建模：

| 核心要素                             | 解释                                       | 结合电话号问题的例子                                         |
| :----------------------------------- | :----------------------------------------- | :----------------------------------------------------------- |
| **1. 路径 (Path)**                   | 你已经做出的选择构成的序列。               | 代码中的 `path` 数组，如 `['a', 'd']`。                      |
| **2. 选择列表 (Choice List)**        | 在当前决策点，你可以选择的所有合法选项。   | 代码中的 `data[int(digits[i])]`，即当前数字对应的字母串，如 `'abc'`。 |
| **3. 结束条件 (Terminal Condition)** | 判断路径是否构成一个完整解，从而停止递归。 | 代码中的 `i == n`，即索引已走到数字串末尾。                  |

你前面看的那段代码，正是这个模型的完美体现。

------

### 三、经典代码框架

掌握下面这个**递归回溯模板**，就抓住了它的骨架：

python

```python
def backtrack(路径, 选择列表):
    if 满足结束条件:
        收集结果(路径)
        return

    for 选择 in 选择列表:
        # 1. 做出选择
        将该选择加入路径
        # 2. 向下递归探索
        backtrack(路径, 新的选择列表)
        # 3. 撤销选择（回溯！）
        从路径中移除该选择
```

### 四、显式回溯 vs. 隐式回溯

你刚刚看的代码有个巧妙之处，它使用了**隐式回溯**，有必要单独指出：

- **显式回溯（标准做法）**：使用 `append` 和 `pop` 等操作，明确地添加和移除路径元素。选择列表通常用剩余候选元素来表示。

  python

  ```python
  path.append(c)      # 做选择
  dfs(...)            # 递归
  path.pop()          # 撤销选择，显式回溯
  ```

  

- **隐式回溯（你的代码）**：通过**索引直接赋值**来修改一个固定大小的数组。

  python

  ```python
  path[i] = c         # 直接覆盖，等同于为位置i做选择
  dfs(i + 1)          # 递归处理下一个位置
  # 没有 pop 操作。下一个循环会直接覆盖 path[i]，自然完成了“撤销”
  ```

  

  这种方式之所以成立，是因为**决策点 `i` 与路径索引是严格对应的**。当从递归返回，循环尝试下一个字符时，`path[i]` 会被新字符直接覆盖，相当于自动撤销了上一步的选择。这在字符串长度固定时是更高效的写法。

### 五、与相关概念的辨析

#### 1. 回溯 vs. 深度优先搜索 (DFS)

- **DFS** 是更广义的图/树遍历算法。
- **回溯** 是 DFS 的一种**特化和应用**。它利用 DFS 遍历解空间树，但核心在于“做选择”和“撤销选择”的**状态重置**动作。所有回溯算法都是 DFS，但并非所有 DFS 都是回溯（比如单纯的二叉树遍历不涉及状态撤销）。

#### 2. 回溯 vs. 动态规划 (DP)

- 两者都用于解决多阶段决策问题。
- **核心区别在于是否求最优结构**：
  - **回溯**：求**所有**可行解。采用穷举，时间复杂度通常是指数级。
  - **动态规划**：求**一个**最优解（或最优解的值）。要求问题有重叠子问题和最优子结构，通过记录状态来避免重复计算，时间复杂度通常是多项式级。

#### 3. 回溯与剪枝 (Pruning)

- **剪枝是回溯的优化手段**。你的代码没有剪枝，因为它需要生成所有组合。
- 但在像 N 皇后这类问题中，一旦中途发现冲突（比如皇后在同一斜线上），就可以**提前终止当前路径的探索**，直接 `return` 或 `continue`，这就是剪枝。它大幅削减了搜索空间，是回溯算法的核心优化思想。

### 六、适用的经典题型

当你看到以下关键词时，应立刻联想到回溯：

- **排列 / 组合 / 子集问题** (Permutations / Combinations / Subsets)
- **切割问题** (Palindrome Partitioning)
- **棋盘问题** (N-Queens, Sudoku Solver)
- **括号生成** (Generate Parentheses)
- **路径查找** (Word Search)

总而言之，回溯算法的精髓就在于 **“向前走一步，记下足迹；若到尽头，则退回一步，擦除足迹，再试新路”**。
