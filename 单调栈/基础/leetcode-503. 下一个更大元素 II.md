# [503. 下一个更大元素 II](https://leetcode.cn/problems/next-greater-element-ii/)

给定一个循环数组 `nums` （ `nums[nums.length - 1]` 的下一个元素是 `nums[0]` ），返回 *`nums` 中每个元素的 **下一个更大元素*** 。

数字 `x` 的 **下一个更大的元素** 是按数组遍历顺序，这个数字之后的第一个比它更大的数，这意味着你应该循环地搜索它的下一个更大的数。如果不存在，则输出 `-1` 。

**示例 1:**

```
输入: nums = [1,2,1]
输出: [2,-1,2]
解释: 第一个 1 的下一个更大的数是 2；
数字 2 找不到下一个更大的数； 
第二个 1 的下一个最大的数需要循环搜索，结果也是 2。
```

**示例 2:**

```
输入: nums = [1,2,3,4,3]
输出: [2,3,4,-1,4]
```

**提示:**

- `1 <= nums.length <= 104`
- `-109 <= nums[i] <= 109`

```python
class Solution:
    def nextGreaterElements(self, nums: List[int]) -> List[int]:
        n = len(nums)
        st =[]
        left = [-1]*2*n
        copy_nums = nums.copy()
        new_nums = nums+copy_nums
        print(new_nums)
        for i,x in enumerate(new_nums):
            while st and new_nums[st[-1]]<x:
                left[st.pop()]=new_nums[i]
            st.append(i)
```

注：针对循环问题可以采用数组拼接的方式处理循环

## 1. 问题描述

给定一个循环整数数组 `nums`，对每个元素 `nums[i]`，找出它的**下一个更大元素**（右侧第一个比它大的数）。

- 由于数组是**循环**的，最后一个元素的下一个更大元素可能在它的左边（即数组开头）。
- 如果不存在这样的元素，则结果对应位置填 `-1`。

------

## 2. 算法思路

### ① 处理循环的技巧：数组拼接

python

```
copy_nums = nums.copy()
new_nums = nums + copy_nums
```



- 将原数组复制一份接在后面，得到长度为 `2n` 的新数组 `new_nums`。
- 这样，原数组中任何一个元素，在其后的 `n-1` 个位置内如果存在更大值，就一定会落在它的右侧不远处。由于数组首尾相接，拼接后的序列就能模拟循环查找的过程：
  例如原数组 `[1,2,1]` 拼接为 `[1,2,1,1,2,1]`，最后一个 `1` 的下一个更大元素是 `2`，在拼接数组的第二个 `2` 处。
- 最后只需取结果数组的前 `n` 个元素，因为后半段是重复的，我们只为原数组的 `n` 个位置计算答案。

### ② 单调栈查找下一个更大元素

python

```
st = []
left = [-1] * 2 * n   # 用 left 存储结果，虽命名为 left，实为“下一个更大元素”数组
for i, x in enumerate(new_nums):
    while st and new_nums[st[-1]] < x:
        left[st.pop()] = new_nums[i]   # 当前元素 x 是栈顶元素的下一个更大元素
    st.append(i)
```



- 栈中存放的是**下标**，且栈内下标对应的值保持**严格递减**（栈底最大，栈顶最小）。
- 当遍历到一个新元素 `x` 时，若 `x` 比栈顶下标对应的值大，说明栈顶的那个元素“找到了”下一个更大值，于是将栈顶弹出，并把对应位置的答案设为 `x`。
- 不断弹出直到栈空或栈顶值 ≥ `x`，然后将当前下标入栈。
- 遍历结束后，栈中剩余的下标都是**没有找到下一个更大元素**的，它们的结果保持初始的 `-1`。

### ③ 取原数组长度部分返回

python

```
return left[:n]
```



因为只关心原数组的 `n` 个元素，拼接数组的后半段只是辅助查找，结果数组前 `n` 个位置即对应原数组的每个元素。

------

## 3. 举例说明

```
nums = [1, 2, 1]`
拼接：`new_nums = [1, 2, 1, 1, 2, 1]
```

遍历过程（下标 i 及值 x）：

- `i=0, x=1`：栈空，入栈 `[0]`
- `i=1, x=2`：栈顶 1 < 2，弹出 0，`left[0]=2`；栈空，入栈 `[1]`
- `i=2, x=1`：栈顶 2 > 1，不弹，入栈 `[1,2]`
- `i=3, x=1`：栈顶 1 不小于 1（相等），不弹，入栈 `[1,2,3]`
- `i=4, x=2`：栈顶 1 < 2，弹出 3，`left[3]=2`；新栈顶 1 < 2，弹出 2，`left[2]=2`；栈顶 2 不小于 2，停止，入栈 `[1,4]`
- `i=5, x=1`：栈顶 2 > 1，不弹，入栈 `[1,4,5]`

最终 `left = [2, -1, 2, 2, -1, -1]`
取前 3 个：`[2, -1, 2]`
解释：原数组 `[1,2,1]` → 第一个 1 下一个更大是 2，2 没有更大，最后一个 1 循环到开头找到 2。

------

## 4. 复杂度分析

- **时间复杂度**：O(n)。遍历 `2n` 个元素，每个元素入栈、出栈最多一次。
- **空间复杂度**：O(n)。`new_nums` 长度 `2n`，栈和结果数组长度均为 `2n`，整体为 O(n) 级别。
