# [739. 每日温度](https://leetcode.cn/problems/daily-temperatures/)

给定一个整数数组 `temperatures` ，表示每天的温度，返回一个数组 `answer` ，其中 `answer[i]` 是指对于第 `i` 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 `0` 来代替。

**示例 1:**

```
输入: temperatures = [73,74,75,71,69,72,76,73]
输出: [1,1,4,2,1,1,0,0]
```

**示例 2:**

```
输入: temperatures = [30,40,50,60]
输出: [1,1,1,0]
```

**示例 3:**

```
输入: temperatures = [30,60,90]
输出: [1,1,0]
```

**提示：**

- `1 <= temperatures.length <= 105`
- `30 <= temperatures[i] <= 100`

## 写法一：从右到左

```Python
class Solution:
    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        n = len(temperatures)
        answer = [0]*n
        st = []
        for i in range(n-1,-1,-1):
             t = temperatures[i]
             while st and temperatures[st[-1]]<=t:
                 st.pop()
             if st:
                 answer[i] = st[-1]-i
             st.append(i)
        return answer

```

###  1.核心思路：从右向左遍历，维护“未来更高温度的下标”单调栈

我们从最后一天向前遍历，同时用一个栈 `st` 来保存**已经遍历过的（即当前下标右侧的）那些天数的下标**，并且确保栈内的温度是**严格递增**的（单调递增栈，栈底到栈顶温度递增，或这里具体说是：栈顶的温度是离当前最近且大于当前温度的天）。

#### 详细步骤：

- 初始化结果数组 `answer = [0] * n`，栈 `st = []`。
- 从 `i = n-1` 倒序遍历到 `0`：
  1. 当前温度 `t = temperatures[i]`。
  2. **弹出栈中所有温度 ≤ 当前温度的下标**：
     因为这些天比当前温度低或相等，对于第 `i` 天来说，它们不可能是“下一个更高温度”，而且对更左边的天（`< i`）来说，当前天 `i` 温度更高且更近，会完全替代这些被弹出的天，所以它们不再有用。
  3. 如果栈非空，那么栈顶的下标就是**右侧第一个温度更高**的天（因为栈内温度递增，栈顶温度最小且 > 当前温度，并且距离最近）。
     计算等待天数：`answer[i] = st[-1] - i`。
  4. 将当前下标 `i` 入栈（此时栈中剩下的都是温度 > 当前温度的天，当前温度入栈后仍保持递增）。

### 2.为什么栈是单调递增的？

栈中存放的是“可能成为未来更高温度”的下标。当我们处理第 `i` 天时：

- 如果栈顶的温度 ≤ 当前温度，那么对于 `i` 左边的天来说，当前温度更高且更近，栈顶的那些天就永远用不到了，因此弹出。
- 剩下的栈顶就是第一个温度 > 当前温度的天，也就是我们需要的答案。
- 将当前下标入栈后，由于之前弹出的都是 ≤ 当前温度的天，现在栈顶是 > 当前温度的天，因此栈内温度从底到顶递增。

```
temperatures = [73, 74, 75, 71, 69, 72, 76, 73]
```

- 从 `i=7` (温度 73) 开始：栈空，`answer[7]=0`，栈 `[7]`。

- `i=6` (76)：栈顶是 73 ≤ 76，弹出。栈空，`answer[6]=0`，栈 `[6]`。

- `i=5` (72)：栈顶 76 > 72，`answer[5] = 6-5 = 1`，栈 `[6,5]`。

- `i=4` (69)：栈顶 72 > 69，`answer[4] = 5-4 = 1`，栈 `[6,5,4]`。

- `i=3` (71)：弹出 69 ≤ 71，弹出 72 > 71 停下，栈顶 72，`answer[3] = 5-3 = 2`，栈 `[6,5,3]`。

- `i=2` (75)：弹出 71 ≤ 75，弹出 72 ≤ 75，弹出 76 > 75 停下，栈顶 76，`answer[2] = 6-2 = 4`，栈 `[6,2]`。

- `i=1` (74)：栈顶 75 > 74，`answer[1] = 2-1 = 1`，栈 `[6,2,1]`。

- `i=0` (73)：栈顶 74 > 73，`answer[0] = 1-0 = 1`，栈 `[6,2,1,0]`。
  最终答案 `[1,1,4,2,1,1,0,0]`，与标准解一致。

  

### 3.复杂度分析

- 时间复杂度：O(n)。每个下标入栈一次、出栈一次，内层 while 循环总体执行次数 O(n)。
- 空间复杂度：O(n)。栈最大可能存储所有下标。





## 写法二：从左到右

```python
class Solution:
    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        n = len(temperatures)
        answer = [0]*n
        st = []
        for idx,temperature in enumerate(temperatures):
            while st and temperatures[st[-1]]<temperature:
                j=st.pop()
                print(j)
                answer[j]=idx-j
                print(answer)
            st.append(idx)
        return answer
```

## 算法思想

### 1. 核心思路：利用单调栈“暂存”还没找到更高温度的天数

- 遍历每天的温度，用栈 `st` 存放**暂时还没有找到下一个更高温度**的天数的下标。
- 栈中下标对应的温度是**递减**的（从栈底到栈顶温度递减），因为如果遇到一个更高的温度，它就会成为栈中所有比它低的天数的“下一个高温”，那些天数就找到了答案，可以从栈中弹出。
- 对于当前天 `idx`，其温度为 `temperature`：
  1. 不断检查栈顶下标 `st[-1]` 对应的温度，如果比当前温度**低**，说明栈顶这一天终于等到了更高的温度，**弹出**栈顶下标 `j`，并计算等待天数为 `idx - j`，存入答案数组 `answer[j] = idx - j`。
  2. 重复弹出，直到栈为空或者栈顶温度 ≥ 当前温度。
  3. 将当前下标 `idx` 压入栈（因为它自己也需要等待未来更高的温度）。
- 遍历完成后，栈中剩余的下标都是“之后没有更高温度”的天数，这些位置的答案保持初始的 `0`，无需处理。

### 2. 为什么栈是单调递减的？

因为：

- 如果遇到一个较高的温度，所有比它低的温度都会“解决”掉（弹出）。
- 剩下的栈顶温度一定 ≥ 当前温度，所以压入当前温度后，栈内温度仍保持递减顺序。
  这种结构确保了每次弹出时，当前天就是栈顶天的**第一个**更高温度，并且是最近的。

### 3. 举例说明

```
temperatures = [73, 74, 75, 71, 69, 72, 76, 73]
```

- `idx=0, temp=73`：栈空，压栈 `[0]`。
- `idx=1, temp=74`：栈顶 73 < 74，弹出 0，`answer[0]=1`；栈空，压栈 `[1]`。
- `idx=2, temp=75`：栈顶 74 < 75，弹出 1，`answer[1]=1`；栈空，压栈 `[2]`。
- `idx=3, temp=71`：栈顶 75 > 71，不弹，压栈 `[2,3]`。
- `idx=4, temp=69`：栈顶 71 > 69，压栈 `[2,3,4]`。
- `idx=5, temp=72`：栈顶 69 < 72，弹出 4，`answer[4]=1`；新栈顶 71 < 72，弹出 3，`answer[3]=2`；栈顶 75 > 72，停止，压栈 `[2,5]`。
- `idx=6, temp=76`：栈顶 72 < 76，弹出 5，`answer[5]=1`；新栈顶 75 < 76，弹出 2，`answer[2]=4`；栈空，压栈 `[6]`。
- `idx=7, temp=73`：栈顶 76 > 73，压栈 `[6,7]`。
  最终 `answer = [1,1,4,2,1,1,0,0]`。

### 4. 时间复杂度

- 每个下标最多入栈一次、出栈一次，内层 `while` 循环总执行次数为 O(n)，整体时间复杂度 **O(n)**。
- 空间复杂度 O(n)（栈的大小）。
