# [1475. 商品折扣后的最终价格](https://leetcode.cn/problems/final-prices-with-a-special-discount-in-a-shop/)

给你一个数组 `prices` ，其中 `prices[i]` 是商店里第 `i` 件商品的价格。

商店里正在进行促销活动，如果你要买第 `i` 件商品，那么你可以得到与 `prices[j]` 相等的折扣，其中 `j` 是满足 `j > i` 且 `prices[j] <= prices[i]` 的 **最小下标** ，如果没有满足条件的 `j` ，你将没有任何折扣。

请你返回一个数组，数组中第 `i` 个元素是折扣后你购买商品 `i` 最终需要支付的价格。

**示例 1：**

```
输入：prices = [8,4,6,2,3]
输出：[4,2,4,2,3]
解释：
商品 0 的价格为 price[0]=8 ，你将得到 prices[1]=4 的折扣，所以最终价格为 8 - 4 = 4 。
商品 1 的价格为 price[1]=4 ，你将得到 prices[3]=2 的折扣，所以最终价格为 4 - 2 = 2 。
商品 2 的价格为 price[2]=6 ，你将得到 prices[3]=2 的折扣，所以最终价格为 6 - 2 = 4 。
商品 3 和 4 都没有折扣。
```

**示例 2：**

```
输入：prices = [1,2,3,4,5]
输出：[1,2,3,4,5]
解释：在这个例子中，所有商品都没有折扣。
```

**示例 3：**

```
输入：prices = [10,1,1,6]
输出：[9,0,1,6]
```

**提示：**

- `1 <= prices.length <= 500`
- `1 <= prices[i] <= 10^3`

```python
#从左到右
class Solution:
    def finalPrices(self, prices: List[int]) -> List[int]:
        n =len(prices)
        left = [-1]*n
        st = []
        for i,num in enumerate(prices):
            while st and prices[st[-1]]>=num:
                prices[st.pop()]-=num
            st.append(i)
        return prices

```

## 1. 问题目标

对于数组 `prices` 中的每个元素，找出它**右侧第一个小于等于它的元素**，并将该元素值作为折扣减去。如果没有这样的元素，则价格不变。
要求**原地修改**并返回最终价格数组。

------

## 2. 算法思想：从左到右维护单调递增栈

我们使用一个栈 `st`，里面存放**还没有找到折扣的商品的下标**，并且保证**栈中下标对应的价格是严格递增的**（栈底最小，栈顶最大）。

遍历数组时，对于当前价格 `num`（下标 `i`）：

- **当前价格可以作为栈中某些元素的折扣**：
  只要栈非空，并且栈顶下标对应的价格 **≥ 当前价格**，说明栈顶元素找到了它右边第一个 ≤ 它的价格（即 `num`），于是将栈顶元素的价格减去 `num`（完成打折），并将其弹出。
- 重复弹出直到栈空或栈顶价格 **< 当前价格**。
- 将当前下标 `i` 入栈（因为它自己也需要继续向右找折扣）。

**为什么栈是递增的？**
因为一旦遇到一个较小的价格，所有之前更大的价格都会被弹出（它们找到了折扣）。剩下的栈顶价格一定小于当前价格，压入当前价格后，栈内价格依然保持递增。

------

## 3. 遍历过程示例

```
prices = [8,4,6,2,3]
```

- i=0, num=8：栈空，压栈 `[0]`
- i=1, num=4：栈顶 prices[0]=8 ≥ 4，弹出0，prices[0] = 8-4=4；继续栈空，压栈 `[1]`
- i=2, num=6：栈顶 prices[1]=4 < 6，不弹，压栈 `[1,2]`
- i=3, num=2：栈顶 prices[2]=6 ≥ 2，弹出2，prices[2]=6-2=4；新栈顶 prices[1]=4 ≥ 2，弹出1，prices[1]=4-2=2；压栈 `[3]`
- i=4, num=3：栈顶 prices[3]=2 < 3，压栈 `[3,4]`

最终 `prices = [4,2,4,2,3]`

------

## 4. 复杂度分析

- **时间复杂度**：O(n)。每个元素入栈一次、出栈一次，内层 while 循环总操作次数为 O(n)。
- **空间复杂度**：O(n)。栈最多存储所有元素的下标。



```python
#从右向左
class Solution:
    def finalPrices(self, prices: List[int]) -> List[int]:
        n =len(prices)
        right = [-1]*n
        st = [0]
        for i in range(n-1,-1,-1):
            num = prices[i]
            while  st[-1]>num:
                st.pop()
            right[i]=num-st[-1]
            st.append(num)
        return right
```

## 1. 问题目标

对于数组 `prices` 的每个元素，找到它 **右侧第一个 ≤ 它的价格**，然后用原价减去这个价格作为折扣后的最终价格；如果右侧没有 ≤ 它的价格，则维持原价。

------

## 2. 算法核心思路：从右向左维护一个递增栈，栈顶即“最近的更小价格”

### 栈的定义与作用

- 栈 `st` 中存放的是**当前位置右侧的所有价格**，并且始终保持**严格递增**（栈底最小，栈顶最大）。
- 这样栈顶就是右侧所有价格中**最小的那个**，也就是**右侧第一个 ≤ 当前价格**的值（因为如果右侧有比当前价格更小或相等的，它一定在栈顶；如果栈顶比当前价格大，则当前价格右侧不存在 ≤ 它的值）。
- **哨兵 `0`**：初始化栈为 `[0]`，确保栈永不为空。当没有符合条件的价格时，栈顶就是 `0`，此时 `num - 0 = num`，正好等于原价，无需特殊处理。

### 算法步骤

1. 初始化结果数组 `right = [-1] * n`，栈 `st = [0]`。
2. 从右向左遍历 `i = n-1` 到 `0`：
   - `num = prices[i]`。
   - **弹出栈中所有 > num 的元素**：因为这些价格比当前价格大，对于更左侧的元素来说，当前价格更小且更近，完全挡住了它们，它们不会再成为任何元素的折扣。
   - 弹出后，栈顶就是 `≤ num` 且距离最近的价格。
   - `right[i] = num - st[-1]`，得到折扣后的价格。
   - 将 `num` 压入栈，栈依然保持递增（因为已弹出所有比它大的，新栈顶就是刚刚压入的 `num`，后面的都比它小）。
3. 返回 `right`。

------

## 3. 示例演示

```
prices = [8, 4, 6, 2, 3]
```

| i    | num  | 弹出前 st | 弹出后 st   | 栈顶 | right[i] | 压栈后 st    |
| :--- | :--- | :-------- | :---------- | :--- | :------- | :----------- |
| 4    | 3    | [0]       | [0]         | 0    | 3-0=3    | [0, 3]       |
| 3    | 2    | [0, 3]    | 弹出3→[0]   | 0    | 2-0=2    | [0, 2]       |
| 2    | 6    | [0, 2]    | [0, 2]      | 2    | 6-2=4    | [0, 2, 6]    |
| 1    | 4    | [0, 2, 6] | 弹出6→[0,2] | 2    | 4-2=2    | [0, 2, 4]    |
| 0    | 8    | [0, 2, 4] | [0, 2, 4]   | 4    | 8-4=4    | [0, 2, 4, 8] |

结果：`[4, 2, 4, 2, 3]`，与预期一致。

------

## 4. 正确性分析

- **单调性**：栈中元素始终递增。当我们遇到一个较小的价格时，所有比它大的候选都被弹出，保证栈顶是第一个 ≤ 当前价格的值。
- **最近性**：由于从右向左遍历，先入栈的元素在原始数组中更靠右；弹出操作保留了最近且更小的那个。
- **哨兵 0**：因为所有价格均为正整数，0 永远 ≤ 任何价格，在没有更小价格时栈顶就是 0，相减得原价，完美适配“无折扣”情况。

------

## 5. 复杂度

- **时间复杂度**：O(n)。每个元素入栈、出栈最多各一次。
- **空间复杂度**：O(n)。栈最深时可能存储所有元素。
