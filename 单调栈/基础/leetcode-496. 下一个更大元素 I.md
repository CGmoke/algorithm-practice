# [496. 下一个更大元素 I](https://leetcode.cn/problems/next-greater-element-i/)

`nums1` 中数字 `x` 的 **下一个更大元素** 是指 `x` 在 `nums2` 中对应位置 **右侧** 的 **第一个** 比 `x` 大的元素。

给你两个 **没有重复元素** 的数组 `nums1` 和 `nums2` ，下标从 **0** 开始计数，其中`nums1` 是 `nums2` 的子集。

对于每个 `0 <= i < nums1.length` ，找出满足 `nums1[i] == nums2[j]` 的下标 `j` ，并且在 `nums2` 确定 `nums2[j]` 的 **下一个更大元素** 。如果不存在下一个更大元素，那么本次查询的答案是 `-1` 。

返回一个长度为 `nums1.length` 的数组 `ans` 作为答案，满足 `ans[i]` 是如上所述的 **下一个更大元素** 。

**示例 1：**

```
输入：nums1 = [4,1,2], nums2 = [1,3,4,2].
输出：[-1,3,-1]
解释：nums1 中每个值的下一个更大元素如下所述：
- 4 ，用加粗斜体标识，nums2 = [1,3,4,2]。不存在下一个更大元素，所以答案是 -1 。
- 1 ，用加粗斜体标识，nums2 = [1,3,4,2]。下一个更大元素是 3 。
- 2 ，用加粗斜体标识，nums2 = [1,3,4,2]。不存在下一个更大元素，所以答案是 -1 。
```

**示例 2：**

```
输入：nums1 = [2,4], nums2 = [1,2,3,4].
输出：[3,-1]
解释：nums1 中每个值的下一个更大元素如下所述：
- 2 ，用加粗斜体标识，nums2 = [1,2,3,4]。下一个更大元素是 3 。
- 4 ，用加粗斜体标识，nums2 = [1,2,3,4]。不存在下一个更大元素，所以答案是 -1 。
```

**提示：**

- `1 <= nums1.length <= nums2.length <= 1000`
- `0 <= nums1[i], nums2[i] <= 104`
- `nums1`和`nums2`中所有整数 **互不相同**
- `nums1` 中的所有整数同样出现在 `nums2` 中



```python
class Solution:
    def nextGreaterElement(self, nums1: List[int], nums2: List[int]) -> List[int]:
        n1=len(nums1)
        ans = [-1]*n1
        st=[]
        idx = {x:i for i,x in enumerate(nums1)}
        for x in nums2:
            while st and x>st[-1]:
                ans[idx[st.pop()]]=x 
            if x in idx:
                st.append(x)   
        return ans


```

## 1. 问题描述

给定两个没有重复元素的数组 `nums1`（是 `nums2` 的子集）和 `nums2`，对每个 `nums1[i]`，找到它在 `nums2` 中对应位置的**右侧第一个更大的元素**，若不存在则返回 `-1`。

------

## 2. 算法思路

### ① 建立映射



```python
idx = {x: i for i, x in enumerate(nums1)}
```



将 `nums1` 的每个值映射到其在 `nums1` 中的下标，方便后续直接修改答案数组。

### ② 单调栈遍历 `nums2`

- 使用一个栈 `st`，里面存放的是 **`nums1` 中已经遇到但尚未找到下一个更大元素的那些值**。
- 栈中的元素保持**严格递减**（栈顶最小），因为一旦遇到一个比栈顶大的值，栈顶元素就“找到”了下一个更大元素，应立即弹出并记录。

### ③ 核心循环



```python
for x in nums2:
    while st and x > st[-1]:
        ans[idx[st.pop()]] = x
    if x in idx:
        st.append(x)
```



- **弹出并记录答案**：当前遍历值 `x` 大于栈顶元素，说明 `x` 就是栈顶元素的下一个更大元素。将栈顶弹出，利用 `idx` 找到它在 `ans` 中的位置，赋值为 `x`。不断弹出，直到栈空或栈顶值 ≥ `x`。
- **入栈**：如果 `x` 是 `nums1` 中的元素（即 `x in idx`），则将它压入栈，等待后续寻找它的下一个更大元素。
  **注意**：如果 `x` 不在 `nums1` 中，我们根本不需要关心它是否找到更大元素，因此不压栈，这样既节省空间，也避免了无意义的比较。

### ④ 最终结果

遍历结束后，栈中剩余的元素都是**没有找到下一个更大元素**的，它们在 `ans` 中保留初始值 `-1`，符合题意。

------

## 3. 举例说明

```python
nums1 = [4,1,2], nums2 = [1,3,4,2]
```

- `idx = {4:0, 1:1, 2:2}`
- 遍历 `x`：
  - `x=1`：栈空，`1` 在 `nums1` 中，压栈 → `st=[1]`
  - `x=3`：`3 > st[-1] (1)`，弹出 `1`，`ans[1]=3`；栈空；`3` 不在 `nums1`，不压栈 → `st=[]`
  - `x=4`：栈空，`4` 在 `nums1` 中，压栈 → `st=[4]`
  - `x=2`：`2` 不大于 `4`，不弹；`2` 在 `nums1` 中，压栈 → `st=[4,2]`（注意栈内递减：4 > 2）

遍历结束，栈中剩余 `4,2` 没有更大值，`ans` 中对应位置保持 `-1`。最终 `ans = [-1, 3, -1]`，正确。

------

## 4. 复杂度分析

- **时间复杂度**：O(len(nums1) + len(nums2))。每个元素至多入栈一次、出栈一次，哈希表查找为 O(1)。
- **空间复杂度**：O(len(nums1))。栈中最多存储 `nums1` 的所有元素，哈希表大小也为 `nums1` 长度。
