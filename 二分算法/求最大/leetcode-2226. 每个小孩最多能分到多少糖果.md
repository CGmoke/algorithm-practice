# [2226. 每个小孩最多能分到多少糖果](https://leetcode.cn/problems/maximum-candies-allocated-to-k-children/)

给你一个 **下标从 0 开始** 的整数数组 `candies` 。数组中的每个元素表示大小为 `candies[i]` 的一堆糖果。你可以将每堆糖果分成任意数量的 **子堆** ，但 **无法** 再将两堆合并到一起。

另给你一个整数 `k` 。你需要将这些糖果分配给 `k` 个小孩，使每个小孩分到 **相同** 数量的糖果。每个小孩可以拿走 **至多一堆** 糖果，有些糖果可能会不被分配。

返回每个小孩可以拿走的 **最大糖果数目** 。

 

**示例 1：**

```
输入：candies = [5,8,6], k = 3
输出：5
解释：可以将 candies[1] 分成大小分别为 5 和 3 的两堆，然后把 candies[2] 分成大小分别为 5 和 1 的两堆。现在就有五堆大小分别为 5、5、3、5 和 1 的糖果。可以把 3 堆大小为 5 的糖果分给 3 个小孩。可以证明无法让每个小孩得到超过 5 颗糖果。
```

**示例 2：**

```
输入：candies = [2,5], k = 11
输出：0
解释：总共有 11 个小孩，但只有 7 颗糖果，但如果要分配糖果的话，必须保证每个小孩至少能得到 1 颗糖果。因此，最后每个小孩都没有得到糖果，答案是 0 。
```

 

**提示：**

- `1 <= candies.length <= 105`
- `1 <= candies[i] <= 107`
- `1 <= k <= 1012`

备注：该题应该注意边界判断以及check()函数时间复杂度优化



```python
class Solution:
    def maximumCandies(self, candies: List[int], k: int) -> int:
        def check(mid):
            total = 0
            for c in candies:
                total += c // mid
            return total
        left = 0
        right = max(candies)+1
        while left+1<right:
            mid=(left+right)//2
            if check(mid)>=k:
                left=mid
            else:
                right=mid
        return left
```

## 算法思想分析：最大化分配糖果数量

这道题的本质是：
有 `n` 堆糖果，每堆数量为 `candies[i]`，要将它们分成**大小完全相同**的若干份（每份糖果数相同），且份数至少为 `k`。
求**每份最多**可以有多少颗糖果。

------

### 1. 问题转化 —— 判定问题

定义一个**判定函数**：

> 给定每份大小 `mid`，能否分出至少 `k` 份？

这很简单：对于每一堆糖果，如果它原本有 `c` 颗，每份大小为 `mid`，那这一堆最多可以分出 `⌊c / mid⌋` 份（向下取整，因为剩下的不够一份就浪费了）。
把所有堆能分出的份数加起来，就是总份数。

python

```python
def check(mid):
    total = 0
    for c in candies:
        total += c // mid
    return total
```



- 若 `check(mid) >= k`，则 `mid` 是一个**可行**的大小。
- 若 `check(mid) < k`，则 `mid` **不可行**。

------

### 2. 单调性 —— 二分搜索的前提

这个判定函数具有很好的**单调性**：

- **如果 `mid` 可行**（即总份数 ≥ `k`），那么任何比 `mid` 小的值 `x` **更可行**（份数只会更多，因为每份变小，能分出的份数增加）。
- **如果 `mid` 不可行**（总份数 < `k`），那么任何比 `mid` 大的值 `y` **更不可行**（份数更少）。

所以，可行性关于 `mid` 是**单调递减**的：
从 0 到很大的数，可行性从“可行”变为“不可行”，存在一个分界点——**最大的可行值**正是我们要求的答案。

------

### 3. 二分查找 —— 开区间模板

代码采用了**开区间二分**，维护：

- `left`：**永远可行**的每份大小
- `right`：**永远不可行**的每份大小
- 最终答案就是 `left`（可行区间的右端点）

#### 初始化边界

python

```python
left = 0
right = max(candies) + 1
```



- `left = 0`：理论上每份为 0 可以分出无穷多份，一定可行（代码中 `mid` 不会真的取到 0，所以安全）。
- `right = max(candies) + 1`：比最大堆的糖果数还多 1，此时任何一堆都不够分一份（`c // right` 全是 0），总份数为 0，绝对不可行。

这样初始就满足了 `left` 可行、`right` 不可行的不变式。

#### 循环迭代



```python
while left + 1 < right:
    mid = (left + right) // 2
    if check(mid) >= k:
        left = mid      # mid 可行，尝试更大的值
    else:
        right = mid     # mid 不可行，缩小范围
```



- 当 `left + 1 == right` 时循环结束，此时 `left` 是最后一个可行值，即**最大可行值**。
- 注意 `mid` 的计算始终 ≥ 1（因为 `left` 至少为 0，`right` 至少为 `max+1`，且 `left+1 < right` 保证 `mid` 不会同时为 0），因此 `check` 中除法安全。

#### 返回结果

```python
return left
```



------

### 4. 正确性与边界细节

- **`right = max(candies) + 1` 为什么不是 `sum(candies)`？**
  如果使用 `sum(candies)`，当 `k=1` 且只有一堆糖果时，`sum(candies)` 本身可能就是可行解（例如 `candies=[1], k=1`，每份 1 颗即可），违反了“右边界不可行”的约定，会导致二分循环不执行，返回错误的 `left=0`。
  用 `max(candies) + 1` 能保证绝对不可行，因为任何一堆都不够分出一份完整的大小。
- **`left = 0` 是否合理？**
  可行，因为 `left` 作为可行边界，0 是永远可分的下界。由于循环中 `mid` 始终 ≥ 1，`check(mid)` 不会遇到除零错误。最终答案可能是 0（当糖果总数不足 `k` 时），此时 `left` 保持 0 不变，二分直接返回 0，符合题意。

------

### 5. 时间复杂度

- 判定函数 `check(mid)` 只需遍历一次数组，O(n)。
- 二分查找的搜索范围是 `[0, max(candies)+1]`，设最大值为 M，二分次数为 O(log M)。
- 总时间复杂度 O(n log M)，对于 n ≤ 10⁵, M ≤ 10⁷ 的题目完全可行。
- 空间复杂度 O(1)，仅使用了常数个变量。

------

### 6. 总结

该算法是典型的**二分答案 + 贪心判定**模式，用于解决“最大化最小值”问题。
核心步骤：

1. **定义判定函数**，计算在给定每份大小下能否满足条件。
2. **验证单调性**，确保可行性随参数单调变化。
3. **设计二分边界**，使左边界可行、右边界不可行，避免漏解或死循环。
4. **二分搜索**，根据判定结果收缩边界，最终得到最大可行解。
