# [2529. 正整数和负整数的最大计数](https://leetcode.cn/problems/maximum-count-of-positive-integer-and-negative-integer/)

给你一个按 **非递减顺序** 排列的数组 `nums` ，返回正整数数目和负整数数目中的最大值。

- 换句话讲，如果 `nums` 中正整数的数目是 `pos` ，而负整数的数目是 `neg` ，返回 `pos` 和 `neg`二者中的最大值。

**注意：**`0` 既不是正整数也不是负整数。

**示例 1：**

```
输入：nums = [-2,-1,-1,1,2,3]
输出：3
解释：共有 3 个正整数和 3 个负整数。计数得到的最大值是 3 。
```

**示例 2：**

```
输入：nums = [-3,-2,-1,0,0,1,2]
输出：3
解释：共有 2 个正整数和 3 个负整数。计数得到的最大值是 3 。
```

**示例 3：**

```
输入：nums = [5,20,66,1314]
输出：4
解释：共有 4 个正整数和 0 个负整数。计数得到的最大值是 4 。
```

**提示：**

- `1 <= nums.length <= 2000`
- `-2000 <= nums[i] <= 2000`
- `nums` 按 **非递减顺序** 排列。

```python
class Solution:
    def maximumCount(self, nums: List[int]) -> int:
        def lower_bound(nums):
        	left = 0
        	right = len(nums) - 1
        	while left <= right:
            	mid = (left + right) // 2
            	if nums[mid] >= 0:      # 中间元素 >= 0，目标位置在左侧（包括mid）
                	right = mid - 1
            	else:                    # 中间元素 < 0，目标位置在右侧
                	left = mid + 1
        	return left                  # left 就是第一个 >= 0 的索引

    	ans = lower_bound(nums)          # 负数个数
    	n = len(nums)
    	zero = nums.count(0)             # 零的个数
    	return max(ans, n - ans - zero)  # 正数个数 = 总长 - 负数 - 零
        
```

### 算法思想

由于数组有序，负数和正数分别聚集在数组的两端，中间可能夹着零。

- **负数**：所有负数位于第一个非负数（即第一个 `>=0` 的元素）的左侧。
- **正数**：所有正数位于最后一个非正数（即最后一个 `<=0` 的元素）的右侧。
- **零**：零出现在中间，可以单独统计。

因此，如果能快速找到第一个 `>=0` 的位置（即负数的个数），以及零的个数，就可以用总长度减去负数和零的个数得到正数的个数。

算法步骤：

1. **二分查找第一个 `>=0` 的位置**（即 `lower_bound` 语义）：
   - 如果 `nums[mid] >= 0`，说明目标位置在 `mid` 或左侧，收紧右边界 `right = mid - 1`。
   - 否则（`nums[mid] < 0`），说明目标位置在右侧，收紧左边界 `left = mid + 1`。
   - 循环结束时，`left` 指向第一个 `>=0` 的索引。这也等于负数的个数（因为索引从 0 开始，左侧全是负数）。
2. **统计零的个数**：直接调用 `nums.count(0)`，由于零也满足 `>=0`，它们位于第一个 `>=0` 的位置及其右侧，但需要单独统计出来以便计算正数个数。
3. **计算正数个数**：总长度 `n` 减去负数个数（`ans`）再减去零的个数（`zero`），剩下的就是正数个数。
4. **返回最大值**：`max(负数个数, 正数个数)`。

### 算法正确性分析

- **二分查找不变量**：循环始终保证 `[0, left-1]` 内的元素都 `< 0`，`[right+1, n-1]` 内的元素都 `>= 0`。初始时区间为空，逐渐收缩。
- **终止条件**：当 `left > right` 时停止，此时 `left` 就是第一个不满足 `< 0` 的位置，即第一个 `>=0` 的位置。
  - 如果所有数都 `< 0`，则 `left` 最终为 `n`，此时负数个数为 `n`，零和正数个数为 0。
  - 如果所有数都 `>= 0`，则 `left` 最终为 `0`，此时负数个数为 0。
- **零的处理**：零的个数通过 `count(0)` 获得。注意：零不是负数也不是正数，必须从总数中排除。
- **最终比较**：返回负数个数与正数个数的较大值，符合题目要求。

### 复杂度分析

- **时间复杂度**：O(log n) 用于二分查找 + O(n) 用于 `count(0)`（最坏情况）。实际上 `count(0)` 是线性扫描，但 Python 的 `list.count()` 底层用 C 实现，且只扫描一次。整体时间复杂度为 O(n)（因为 `count` 必须遍历整个数组）。可以优化为第二次二分查找来寻找零的边界，使总复杂度降为 O(log n)，但本题 n 规模不大，O(n) 也可接受。
- **空间复杂度**：O(1)，仅使用了常数个变量。

###  优化讨论（可选）

如果想实现真正的 O(log n)，可以用二分查找分别找到：

- 第一个 `>=0` 的位置 → 负数个数
- 第一个 `>0` 的位置 → 正数起始索引 → 正数个数 = n - pos_first_positive
- 零的个数 = (第一个 `>0` 的位置) - (第一个 `>=0` 的位置)

这样只需要两次二分查找，不需要 `count(0)`。但当前解法简洁易懂，已能满足题目要求。

### 在 Python 中，统计列表中某个元素出现的个数

### 1. 使用 `list.count()` 方法（最简单）

python

```python
my_list = [1, 2, 3, 2, 2, 4, 5]
count = my_list.count(2)
print(count)  # 输出 3
```



### 2. 使用 `collections.Counter`（适合统计多个元素）

python

```python
from collections import Counter
my_list = [1, 2, 3, 2, 2, 4, 5]
counter = Counter(my_list)
print(counter[2])  # 输出 3
```



### 3. 手动循环统计

python

```python
my_list = [1, 2, 3, 2, 2, 4, 5]
target = 2
count = 0
for item in my_list:
    if item == target:
        count += 1
print(count)  # 输出 3
```



### 性能对比

- `list.count()` 在底层用 C 实现，对单个元素统计效率很高。
- 如果需要统计多个不同元素的出现次数，`Counter` 只需要遍历一次列表，比多次调用 `count()` 更高效。
- 手动循环可读性稍差，一般没必要。

### 注意事项

- 元素可以是任意可哈希类型（整数、字符串、元组等）。
- 如果元素是列表等可变类型（不可哈希），`Counter` 会报错，此时只能用 `count()` 或循环。
